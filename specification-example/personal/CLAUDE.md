# 个人 Claude 配置模板

> **使用说明：** 将本文件内容合并到 `~/.claude/CLAUDE.md`。
> 个人配置优先级最高，但不应违背团队核心原则。
> 仅配置真正属于个人偏好的内容，项目和团队的规范不在此重复。

---

## 工作风格偏好

### 回复风格

> [个人填写] 你希望 Claude 如何与你沟通，例如：
> - 回复简洁，直接给出答案，不需要大段解释
> - 代码修改优先展示 diff，而非完整文件
> - 遇到模糊需求，先列出我的理解和假设，再动手

### 操作确认偏好

> [个人填写] 哪些操作需要 Claude 先确认再执行，例如：
> - 删除任何文件前必须确认
> - 执行数据库 DDL 前必须展示完整 SQL 并等待确认
> - git push 前必须展示将要推送的 commits

### 代码风格偏好

> [个人填写] 个人的代码风格偏好（在团队规范允许范围内），例如：
> - 注释使用中文（项目规范允许时）
> - 变量命名倾向于完整词汇，不缩写
> - 函数实现不超过 30 行，超过时提示我重构

## 常用工具和环境

### 开发环境

> [个人填写] 你的本地开发环境信息，帮助 Claude 给出更准确的建议，例如：
> - 操作系统：macOS 14 (Apple Silicon)
> - Shell：zsh + Oh My Zsh
> - 编辑器：Neovim / VSCode（Claude Code 使用）
> - Docker Desktop 版本：4.x

### 常用工具别名

> [个人填写] 你的 shell 别名，让 Claude 知道你的快捷命令，例如：
> ```bash
> alias k='kubectl'
> alias d='docker'
> alias g='git'
> alias mk='make'
> ```

### 语言环境

> [个人填写] 偏好使用的语言环境，例如：
> - 技术文档阅读：英文优先
> - Claude 回复语言：中文（代码注释除外）
> - 错误信息和日志保持英文

## 个人工作流

### Git 工作流偏好

> [个人填写] 例如：
> - 分支命名：`feat/<ticket-id>-<short-desc>` / `fix/<ticket-id>-<short-desc>`
> - commit message 使用 Conventional Commits 格式
> - 不使用 `git add .`，总是手动选择要暂存的文件

### 调试偏好

> [个人填写] 例如：
> - 先看错误日志全文，再提出假设
> - 优先使用 printf/log 调试，而非 debugger（性能问题除外）
> - 每次假设只验证一个变量

### AI 辅助使用边界

> [个人填写] 你希望 Claude 主动做什么，不做什么，例如：
> - 可以主动建议重构，但实现前需要我确认
> - 不要主动添加我没有要求的功能
> - 发现代码中的 bug 时，即使不在当前任务范围内也要指出（但不要自动修复）

## 跨项目通用配置

### 默认语言规范引用

> [个人填写] 如果你主要使用某种语言，可以在个人层引用对应规范：
> ```
> 我主要使用 Go 开发。在所有项目中，如果项目规范未指定，
> 默认参考 ~/team-standards/docs/standards/coding/go.md。
> ```

### 个人常用 Prompt 模板

> [个人填写] 你经常用到的 prompt 模式，可以在这里定义为快捷参考：

**Code Review 场景**
```
请 Review 我的改动，重点关注：
1. 错误处理是否完整
2. 是否有并发安全问题
3. 是否有更简洁的实现方式
```

**调试场景**
```
帮我调试这个问题。请先列出你的假设（1-3个），
然后我们逐一验证，而不是直接给出解决方案。
```

**设计讨论场景**
```
我需要实现 <功能>，先帮我分析设计方案，
列出 2-3 个选项和各自的 trade-off，不要直接写代码。
```
