# 为指定代码生成单元测试

请为以下目标生成单元测试：$ARGUMENTS

## 执行步骤

1. 读取目标文件（如果 $ARGUMENTS 是文件路径）或理解目标功能描述
2. 识别需要测试的函数/方法及其依赖关系
3. 生成测试文件，放在与被测文件相同目录下

## 测试要求

参照 `docs/standards/testing.md` 和对应语言规范：

### 覆盖场景（必须包含）

- **正常路径**：典型输入，验证预期输出
- **边界条件**：空值、零值、最大值、最小值
- **错误路径**：无效输入、依赖失败、权限不足
- **并发场景**（如函数涉及并发）

### 代码规范

- 使用 Table-driven test（测试用例集中定义，循环执行）
- 测试函数命名：`Test<FuncName>_<场景描述>`
- 每个测试用例有清晰的 `name` 字段说明场景
- 使用接口 Mock 外部依赖（数据库、HTTP、消息队列）
- 不 Mock 被测对象内部的业务逻辑

### 语言特定要求

- **Go**：使用 `testify/assert`，测试文件以 `_test.go` 结尾，同包放置
- **Python**：使用 `pytest`，文件命名 `test_<module>.py`
- **TypeScript**：使用 `vitest`，文件命名 `<module>.test.ts`

## 输出说明

- 生成完整可运行的测试文件
- 如需要 Mock 但项目尚未有 Mock 基础设施，说明需要补充的依赖
- 测试写完后，提示运行命令（如 `go test ./internal/service/... -v`）让我验证
