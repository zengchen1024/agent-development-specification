# 对当前改动进行全面代码 Review

请对当前工作区的 git 改动（`git diff HEAD`）进行代码 Review。

## Review 步骤

1. 先执行 `git diff HEAD` 获取所有改动内容
2. 按以下维度逐项检查，每个维度给出具体评价（通过 / 有问题 / 建议改进）
3. 汇总发现的问题，按严重程度（必须修复 / 建议修复 / 可选优化）分类

## 检查维度

### 正确性

- 是否有明显的逻辑错误
- 边界条件（空值、零值、极大值）是否处理
- 并发场景是否安全（竞争条件、死锁风险）
- 错误是否被正确处理，而非静默忽略

### 编码规范

- 参照 `docs/standards/coding/base.md`：命名、注释、函数大小
- 参照对应语言规范（`docs/standards/coding/<lang>.md`）的特有要求
- 是否有重复代码，是否可以提取复用

### 安全

- 参照 `docs/standards/security.md`
- 用户输入是否经过验证
- 是否有硬编码密钥、敏感信息
- 日志是否可能泄露敏感数据

### 测试

- 参照 `docs/standards/testing.md`
- 新增代码是否有对应的测试
- 测试是否覆盖了正常路径、错误路径、边界条件

### 可维护性

- 代码意图是否清晰，复杂逻辑是否有注释
- 是否引入了不必要的复杂度
- 是否有 TODO/FIXME 未处理

## 输出格式

```
## 代码 Review 报告

### 总体评价
<一段整体评价>

### 必须修复
- [ ] <文件:行号> <问题描述>

### 建议修复
- [ ] <文件:行号> <问题描述>

### 可选优化
- <文件:行号> <改进建议>

### 通过项
- <维度>：<简要说明>
```
